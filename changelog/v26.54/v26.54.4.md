# v26.54.4

> ⚠️ **本版本系使用金诚同达内部 API 完成的代码质量与安全性专项整改。**

---

## 安全修复

### 供应链安全
- **ONNX 模型下载版本锁定**：`apps/image_rotation/services/orientation/onnx_service.py` 中 `hf_hub_download()` 调用增加 `revision="v1.0"`，防止恶意模型文件被自动拉取执行

### SSRF 防护
- **ICS URL Provider 禁止跟随重定向**：`apps/reminders/services/calendar_providers/ics_url_provider.py` 的同步/异步方法均将 `follow_redirects=True` 改为 `False`，并在收到重定向响应后对目标 URL 二次校验，阻断内网地址访问的绕过路径

### XSS 防护
- **主仓库 Admin HTML 转义**：`apps/documents/services/template/folder_template/admin_service.py` 中 `render_structure_tree()` 渲染的 `name` 字段添加 `django.utils.html.escape()`，防止通过文件夹模板名称注入脚本
- **Plugins 子模块 message_hub Admin 转义**：`plugins/message_hub/admin/inbox_message_admin.py` 中纯文本正文（`body_text`）渲染进 `mark_safe` 之前添加 `html.escape()` 转义

## 性能优化

### HTTP 客户端连接复用
- **POI Service 客户端复用连接**：`apps/core/services/poi_client.py` 中的 `POIServiceClient` 改用实例级别 `httpx.Client` / `httpx.AsyncClient` 缓存，替代每次请求新建客户端的模式，减少 TCP 连接开销

### 集合推导式
- **C403 自动修复**：`plugins/court_automation/guarantee/helpers.py` 与 `plugins/message_hub/services/court/court_fetcher.py` 中 `set([x async for x in ...])` 改写为 `{x async for x in ...}`，避免中间列表分配

## 架构健壮性

### 线程安全
- **BrowserService 单例双重检查锁**：`apps/core/services/browser/service.py` 将 `__new__` 的 naive 实现替换为带 `threading.Lock()` 的双重检查锁，消除多线程环境下的竞态条件

### 资源管理
- **WebDAVProvider 上下文管理器**：`apps/core/cloud_storage/webdav_provider.py` 增加 `__enter__` / `__exit__` 方法，确保 `requests.Session` 在使用后可靠关闭，防止 TCP 连接池泄漏

### 限流中间件
- **Token Rate Limit TOCTOU 说明**：`apps/core/middleware/token_rate_limit.py` 中添加架构注释，明确 `cache.add` + `cache.incr` 在 Gunicorn 单进程部署下的竞态窗口可控；未来如需严格原子性可替换为 Redis INCR

## 代码规范

### ruff 清理
- 移除主仓库 **78 个**过期的 `# noqa: F401` 注释（RUF100）
- 移除 Plugins **9 个**过期的 `# noqa: F401` 注释
- 修复主仓库与 Plugins 共 **2 处** `RUF043`：pytest.raises 的 `match` 参数使用 `re.escape()` 替代原生字符串，消除中文正则误报
- 修复主仓库 **2 处** `E702`：测试代码中分号分隔的多条语句拆分为独立行

### datetime 升级
- **UP017 迁移**：主仓库与 Plugins 共 **5 处** `timezone.utc` 替换为 `datetime.UTC`（Python 3.12 推荐写法）

## 类型系统

### mypy 配置清理
