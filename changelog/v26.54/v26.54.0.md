### JTN OA 盖章申请自动化

通过 Playwright 自动化完成 JTN OA 系统的所函盖章申请，支持 Mac 端本地文件一键发起。

**核心流程：** 用户提供本地文件路径 → 反查关联合同 → 获取 OA 案件编号 → 启动有头浏览器 → 自动登录（复用缓存 cookies 或 SSO 扫码）→ 搜索案件并回填 → 填写盖章表单（文档类型、盖章类型、份数）→ 上传文件 → 保存提交。

**关键实现：**

- **文件路径 → 合同反查**（`StampLookupService`）：取文件所在目录的所有父路径，用 `IN` 查询 `CaseFolderBinding` / `ContractFolderBinding`，一次 DB 查询命中，无需逐级查找。
- **Playwright 盖章自动化**（`PlaywrightStampMixin`）：导航到 `officedocreg.aspx`，案件搜索弹窗是 iframe（`searchProject.aspx`），`#project_no` 是 readonly 需 JS 去掉 readonly 后填值再调用 `fnseach()`；"选择"按钮在 layui layer 层（父页面），不在 iframe 内。
- **文件上传**：上传弹窗是 `legalup/attachment/attachment.html` iframe，上传按钮是 `<div class="uploadBtn disabled">`，需 JS 移除 `disabled` class 后直接 `click()`（`get_by_text` 无法定位动态渲染文字的 div）。
- **后台异步执行**（`ScriptExecutorService.execute_stamp`）：创建 `StampSession` 后提交到 `ThreadPoolExecutor`，线程内用 `asyncio.run()` 调度 Playwright 异步脚本。

**新增文件：**
- `apps/oa_filing/services/oa_scripts/jtn/stamp/` — Playwright 自动化脚本（constants、playwright_stamp、service、stamp_models）
- `apps/oa_filing/services/stamp_lookup_service.py` — 文件路径反查服务
- `apps/oa_filing/models/stamp_session.py` — `StampSession` 模型
- `apps/oa_filing/schemas/stamp_schemas.py` — API 请求/响应 schema
- `apps/oa_filing/api/stamp_api.py` — 三个 API 端点
- `apps/oa_filing/admin/stamp_session_admin.py` — Django admin 注册

**API 端点：**
- `POST /api/v1/oa-stamp/lookup` — 根据文件路径反查合同，返回 OA 案件编号
- `POST /api/v1/oa-stamp/apply` — 发起盖章申请（异步执行，返回 session ID 供轮询）
- `GET /api/v1/oa-stamp/session/{id}` — 查询盖章申请进度
