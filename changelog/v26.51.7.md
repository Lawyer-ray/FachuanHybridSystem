# v26.51.7 - 2026-06-06

## 后端

### 新功能

- **合同格式规范化 - LLM 驱动的智能段落分析**：用 LLM 批量分析段落结构（层级分类 + 手动编号前缀检测），支持多轮验证低置信度段落，替代旧的正则硬编码方案
  - `llm_helper.py`：`ContractStructureAnalyzer.analyze_document()` 一次性分析整个文档，返回 level/prefix/confidence；失败自动 fallback 到基本结构检测
  - `docx_format_normalizer.py`：完全重写为 `DocxFormatNormalizer`，支持从参考文档精确复制格式（页边距、编号定义、页眉页脚含 PAGE 域代码、字体字号）
  - 支持多级编号剥离（`1.2.3.` 格式）、前导全角空格处理、半角/全角括号混用
- **合同格式规范化 - 参考文档自动匹配**：`execute_view` 自动从 `~/Downloads/验收/` 查找与测试文档名称匹配的参考文档（文件名前缀匹配）
- **合同格式规范化 - 异步执行**：格式化任务在后台线程执行，页面立即返回不再卡住；列表页显示"处理中"状态
- **合同格式规范化 - 端到端测试**：`test_format_e2e.py` 对比 3 组测试集/验证集文档的格式差异（页边距、编号、字体、前缀残留），全部通过
- **Python-POI 桥接层**：`contract_format_service.py` + `poi_client.py` 自动检测 POI 服务可用性，优先走 POI，不可用时 fallback 到 Python 格式化

### 修复

- **Django 启动失败修复**：`test_service.py` 添加 `from typing import Optional`，修复 `NameError`
- **LLM 后端选择优化**：`ContractStructureAnalyzer` 默认使用 `None`（自动选择可用后端），不再硬编码 `siliconflow`，支持 fallback 到 `openai_compatible`
- **LLM 超时控制**：批量分析添加 120 秒超时 + 3 次重试机制

### 优化

- **格式规范化上传页面**：拖拽上传、编号类型选择（中文/数字）、AI 辅助开关、模型选择（mimo2.5pro/本地模型）
- **格式规范化列表页面**：批量格式化/删除、状态显示（待处理/处理中/已完成/失败）、下载/重新格式化按钮

## Java (poi-service)

### 新功能

- **ContractFormatService**：完整的合同 DOCX 格式化服务
  - A4 页面布局（可配置页边距）
  - 自动编号系统（中文编号：一、二、三 → 1. 2. 3. → (1) (2) (3)；数字编号：1. → 1.1. → 1.1.1.）
  - 段落格式化（字体、字号、粗体、行距、缩进、对齐）
  - 表格边框标准化
- **ParagraphClassifier**：正则规则 + 启发式算法识别段落类型（TITLE/HEADING_L1-L3/CLAUSE/BODY/SIGNATURE/TABLE）
- **ContractFormatConfig**：196 行配置类，涵盖所有格式参数
- **ContractController**：REST 接口 `POST /api/documents/contract/format`

### 优化

- **TemplateService**：重构模板渲染逻辑，支持占位符服务注册机制
