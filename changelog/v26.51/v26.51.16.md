# v26.51.16 - 2026-06-09

## 后端

### 新功能
- **社交登录框架（Social Auth）**：新增通用 OAuth/社交登录框架，支持多 Provider 插件式接入
  - 新增 `apps.social_auth` 模块，包含 Provider 注册表、SocialAccount 关联表、临时授权码中转机制
  - Provider 抽象层：`SocialProvider` ABC（`get_authorization_url` / `exchange_code` / `get_profile`），新增 Provider 只需 1 个文件 + settings 配置
  - `SocialAccount` 模型：关联用户与社交平台（`unique_together provider + provider_uid`），支持一个用户绑定多个 Provider
  - `TempAuth` 模型：SPA OAuth 回调的安全中转机制（UUID + 5 分钟过期）
  - 通用回调路由：`/social/{provider}/login/` + `/social/{provider}/callback/`，自动分发到正确 Provider
  - API 端点：`GET /api/v1/social/providers`（Provider 列表）、`POST /api/v1/social/token-exchange`（临时码换 JWT）

### 新增
- **微信扫码登录 Provider**：首个社交登录实现（`providers/wechat.py`）
  - 微信开放平台 OAuth2.0 网站应用扫码登录
  - 首次扫码自动创建用户（`is_active=True`，无需审批），用户名 `soc_{openid后12位}`
  - 二次扫码通过 SocialAccount 自动关联已有用户
  - 支持 state CSRF 防护 + 临时码过期校验

## 前端

### 新增
- **SocialLoginButtons 组件**：通用 Provider 按钮容器，从后端 API 动态获取 Provider 列表渲染
- **SocialCallbackPage 页面**：社交登录回调中间页，接收临时授权码换取 JWT token 并完成登录
- **社交登录 feature 模块**：`features/social-auth/`（types、api、components）

### 修改
- **LoginPage**：登录表单下方新增社交登录按钮区域（分隔线 + Provider 按钮）
- **路由配置**：新增 `/social-callback` 路径用于回调页面
