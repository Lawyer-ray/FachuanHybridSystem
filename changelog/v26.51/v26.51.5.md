# v26.51.5 - 2026-06-05

## 后端

### 新功能

- **CourtSMS 管理页"推荐关联案件"聚合搜索**：在 CourtSMS change page 的关联信息字段组下新增推荐区域，页面加载时自动执行四路 OR 查询（案号精确/前缀匹配、法院名称匹配、当事人匹配），按相关度评分排序以卡片形式展示，用户点击即可填充 case 字段
  - 新建 `CourtSMSRecommendationService`（`services/sms/`），包含法院名称提取（CourtDocument.c_fymc > 短信内容正则 > 案号法院代码）、多维度查询、评分排序逻辑
  - 新增 `recommendations_ajax` AJAX 端点（`/admin/automation/courtsms/{id}/recommendations/`）
  - 前端通过 Select2 API 集成 case 字段，点击"选择此案件"自动填充
- **案件和合同 Admin 行级权限控制**：在 `CaseAdmin` 和 `ContractAdmin` 的 `get_queryset` 中接入已有的 `CaseAccessPolicy` / `ContractAccessPolicy`，非管理员用户只能看到自己团队相关的案件和合同
  - 新增 `apply_admin_access_filter` 通用胶水函数（`core/security/admin_access.py`）
  - 复用 API 层权限策略：团队分配 + CaseAccessGrant 显式授权 + is_admin 放行

### 测试

- **Admin 行级权限控制端到端测试**：16 个测试用例覆盖 6 个场景（admin 看全部、团队律师看团队记录、无分配看空列表、CaseAccessGrant 授权叠加、合同案件路径过滤、perm_open_access 全局放行）
