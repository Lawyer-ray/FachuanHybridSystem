## 缓存键变更说明（上线/回滚注意事项）

### 变更内容
- automation token/credentials/account_stats 相关缓存键：对 `ENVIRONMENT_TYPE` 与 `site_name` 做了规范化（清洗字符集、限长、必要时附加哈希）。
- court_token（法院 TokenService）相关缓存键：对 `site_name` 做了相同规范化。

### 运行影响
- 发布后旧缓存键不会被新代码命中，会自然失效（预期行为）。短时间内可能表现为：
  - Token 缓存命中率下降
  - 自动登录/获取 token 调用次数上升
- 这属于“以安全换取一次性缓存抖动”，通常在缓存重新填充后恢复正常。

### 上线建议
- 建议在发布窗口观察：
  - token 获取成功率/失败率
  - token 获取耗时（以及是否触发更多登录/验证码流程）
- 如使用 Redis 缓存，可按命名空间前缀清理本服务相关 key（避免误清其它服务）。当前实现已默认按命名空间清理。

### 回滚建议
- 若回滚到旧版本：旧版本会继续使用旧缓存键，新版本写入的新键同样不会被旧版本命中。\n+- 回滚并不会自动清理新键；如需回收空间，建议用“命名空间前缀清理”策略删除。
