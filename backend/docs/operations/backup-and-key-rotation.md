# 备份恢复与密钥轮换演练验收

本文件定义“数据与密钥可恢复”的验收标准：有目标、有流程、有演练。

## 1. 目标（必须明确）

| 指标 | 含义 | 目标值（示例） |
|---|---|---|
| RPO | 允许丢失的数据时间窗口 | ≤ 15 分钟 |
| RTO | 允许不可用的恢复时间 | ≤ 60 分钟 |

验收标准：RPO/RTO 必须与业务方确认，并写进事故等级定义。

## 2. 备份策略（最低要求）

- 数据库：全量 + 增量（或 WAL/binlog），加密存储
- 对象存储/文件：版本化或可追溯快照
- 配置：只备份非敏感配置；敏感配置走密钥系统（不落库或加密落库）

## 3. 恢复流程（Runbook 模板）

1. 宣布事故等级与冻结写入（必要时）
2. 校验最近可用备份（完整性/可解密）
3. 执行恢复（数据库优先，文件其次）
4. 运行一致性校验（关键表行数、关键索引、关键业务抽样）
5. 逐步放量恢复流量（灰度/限流）
6. 复盘：根因、修复项、门禁项、演练改进项

## 4. 密钥与凭证轮换（最低要求）

### 4.1 分级

- 加密密钥（用于数据加密/密文解码）
- 第三方凭证（API token/app secret）
- 内部签名密钥（JWT/签名）

### 4.2 轮换策略

- 必须支持双写/双验窗口（新旧并存）
- 轮换必须可回滚（快速切回旧 key）
- 轮换必须可审计（谁执行、何时、影响范围）

## 5. 演练与验收

验收标准：

- 至少每季度一次恢复演练，输出演练报告
- 至少每半年一次密钥轮换演练，输出演练报告
- 演练必须记录：耗时、问题、改进项、下次演练日期
